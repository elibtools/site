1. Information We Collect

The application may collect the following categories of information.

1.1 Information You Provide

The app allows users to open, edit, convert, or manage PDF documents stored on their device using advanced tools (Merge, Split, Watermark, etc.).

These files are:

  • Selected by the user manually through the file picker interface.
  • Processed locally on the device using embedded libraries (including PDFBox).
  • Not uploaded to any external server.

Examples include:

  • PDF documents for structural editing.
  • Images selected for high-definition PDF conversion.
  • Text/Images entered for watermarking or metadata management.

The application does not store or transmit these files externally.

2. File Access and Storage

To provide its professional suite functionality, the app requests specific permissions including READ_EXTERNAL_STORAGE, WRITE_EXTERNAL_STORAGE, and MANAGE_EXTERNAL_STORAGE (where applicable for Android versions).

These permissions allow the app to:

  • Open and render PDF files within the Viewer.
  • Save processed files (Compressed, Merged, or Converted) to your storage.
  • Manage media files for Image-to-PDF tasks.
  • Share generated files via system intents.

All file processing occurs locally on the device. No document content is transmitted to the developer or third-party servers.

3. Local Data Storage

The application may store limited data locally on the device using SharedPreferences and internal app storage.

This data may include:

  • App settings and UI preferences.
  • Premium feature status (Pro, Ads-free, or Batch access).
  • Temporary cache for PDF processing.
  • Tool-specific logic (e.g., custom split ranges or watermark styles).

This information remains strictly on the user's device.

4. Advertising

The application uses Google AdMob to display advertisements, including Rewarded Ads to unlock premium features.

AdMob may collect certain information automatically to serve relevant ads:

  • Device information and Advertising ID (AD_ID).
  • Interaction with advertisements.

For more information see:

Google Privacy Policy: https://policies.google.com/privacy

Users can reset or limit advertising tracking in their device settings.

5. Analytics

The application uses Google Firebase Analytics to understand how users interact with the professional tools.

Firebase Analytics may collect information such as:

  • Tool usage statistics (e.g., which PDF tools are most used).
  • Device type and OS version.
  • Session duration and retention.

This information helps improve the application experience. Firebase Analytics does not collect personally identifiable information.

6. Crash Reporting

The application uses Firebase Crashlytics to detect and fix technical issues.

Crash reports may include:

  • Device state and logs at the time of a crash.
  • App version (e.g., v2.4).

No personal documents, file content, or sensitive PDF data are ever included in crash reports.

7. In-App Purchases & Rewarded Features

The application offers premium functionality (Pro, Ads, and Batch tiers) using Google Play Billing.

  • Purchases are processed securely by Google Play.
  • Rewarded Ads allow temporary access to premium tools.
  • The application does not collect or store credit card or payment information.

8. Third-Party Services

The application integrates the following services:

  • Google AdMob (Advertising)
  • Google Firebase Analytics (Performance Monitoring)
  • Google Firebase Crashlytics (Stability)
  • Google Play Billing (Subscriptions/One-time purchases)

9. Data Security

The application implements high-performance security measures to protect user data. Because elib PDF is an offline-first suite, all PDF manipulation occurs locally. Your documents never leave your device.

10. Children's Privacy

This application is not directed toward children under the age of 13. We do not knowingly collect personal information from children.
